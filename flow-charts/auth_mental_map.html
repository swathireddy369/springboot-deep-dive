üö® JWT Security Challenges

1. Token Theft

Threat: If stolen, attacker can call APIs until expiry

Mitigation: HTTPS only + HttpOnly, Secure cookies

2. No Easy Revocation

Threat: Can't "delete" JWT on server easily

Mitigation: Short-lived tokens (15 min) + refresh tokens + JTI blacklist

3. Large Payloads

Threat: Bloated HTTP requests, header size limits

Mitigation: Keep payload minimal (just userID, roles)

4. Algorithm Confusion

Threat: Clients might force "none" algorithm

Mitigation: Hard-code accepted alg (HS256/RS256) on server
